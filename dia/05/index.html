Unique Paths

Mayo 05

El algoritmo Unique Paths es un algoritmo que se utiliza para resolver el problema de encontrar el número de caminos únicos desde la esquina superior izquierda hasta la esquina inferior derecha de una cuadrícula (o matriz) m x n, en el que solo se permiten movimientos hacia abajo o hacia la derecha.

El algoritmo se basa en la técnica de programación dinámica, que es un método para resolver problemas complejos dividiéndolos en subproblemas más pequeños y luego resolviéndolos en orden ascendente de complejidad. En este caso, se divide el problema de encontrar el número de caminos únicos en subproblemas más pequeños y se utiliza la solución a cada subproblema para construir la solución del problema original.

El algoritmo Unique Paths utiliza una matriz de tamaño m x n para almacenar los resultados de los subproblemas. La entrada en la posición (i, j) de la matriz representa el número de caminos únicos desde la esquina superior izquierda hasta la posición (i, j) de la cuadrícula. Para calcular el número de caminos únicos desde la posición (i, j), se suma el número de caminos únicos desde la posición inmediatamente superior (i-1, j) y el número de caminos únicos desde la posición inmediatamente a la izquierda (i, j-1).

El algoritmo comienza inicializando la primera fila y la primera columna de la matriz con el valor 1, ya que solo hay una forma de llegar a cualquier posición de estas filas y columnas (solo se pueden hacer movimientos hacia la derecha o hacia abajo). A continuación, se recorre la matriz por filas y por columnas, calculando el número de caminos únicos para cada posición de la cuadrícula.

Una vez que se ha recorrido toda la matriz, el resultado se encuentra en la posición (m, n) de la matriz, que representa el número de caminos únicos desde la esquina superior izquierda hasta la esquina inferior derecha de la cuadrícula.

El algoritmo Unique Paths tiene una complejidad temporal de O(mn) y una complejidad espacial de O(mn), lo que lo hace muy eficiente para resolver problemas de tamaño moderado.

Ejemplo

Supongamos que tenemos una cuadrícula de 3 x 3, como se muestra a continuación:

S - - 
- - - 
- - D

Donde "S" representa la esquina superior izquierda y "D" representa la esquina inferior derecha. El objetivo es encontrar el número de caminos únicos que existen para llegar de "S" a "D", moviéndose solo hacia abajo o hacia la derecha.

Para resolver este problema utilizando el algoritmo Unique Paths, se crearía una matriz de 3 x 3, como se muestra a continuación:

1 - - 
- - - 
- - -

La primera fila y la primera columna se inicializan con el valor 1, ya que solo hay una forma de llegar a cualquier posición de estas filas y columnas (solo se pueden hacer movimientos hacia la derecha o hacia abajo).

A continuación, se recorre la matriz por filas y por columnas, calculando el número de caminos únicos para cada posición de la cuadrícula. Por ejemplo, para la posición (2, 2), el número de caminos únicos sería la suma del número de caminos únicos desde la posición inmediatamente superior (1, 2) y el número de caminos únicos desde la posición inmediatamente a la izquierda (2, 1).

Una vez que se ha recorrido toda la matriz, el resultado se encuentra en la posición (3, 3) de la matriz, que representa el número de caminos únicos desde la esquina superior izquierda hasta la esquina inferior derecha de la cuadrícula.

En este ejemplo, el número de caminos únicos se desconoce, ya que no se ha resuelto el problema. Pero utilizando el algoritmo Unique Paths, se puede encontrar la solución de manera eficiente.

Caso practico

¡Por supuesto! Supongamos que tienes una cuadrícula de tamaño 4 x 4, como se muestra a continuación:

S - - -
- X - -
- - X -
- - - D

Donde "S" representa la esquina superior izquierda y "D" representa la esquina inferior derecha. Las posiciones marcadas con "X" son obstáculos y no se pueden atravesar.

El objetivo es encontrar el número de caminos únicos que existen para llegar de "S" a "D", moviéndose solo hacia abajo o hacia la derecha y evitando los obstáculos.

Para resolver este problema utilizando el algoritmo Unique Paths, se crearía una matriz de 4 x 4, como se muestra a continuación:

1 - - - 
- 0 - - 
- - 0 - 
- - - 0

La primera fila y la primera columna se inicializan con el valor 1, ya que solo hay una forma de llegar a cualquier posición de estas filas y columnas (solo se pueden hacer movimientos hacia la derecha o hacia abajo). Las posiciones marcadas con "X" se inicializan con el valor 0, ya que no se pueden atravesar.

A continuación, se recorre la matriz por filas y por columnas, calculando el número de caminos únicos para cada posición de la cuadrícula. Para las posiciones que representan obstáculos, el valor se mantiene en 0. Por ejemplo, para la posición (2, 2), el número de caminos únicos sería la suma del número de caminos únicos desde la posición inmediatamente superior (1, 2) y el número de caminos únicos desde la posición inmediatamente a la izquierda (2, 1), ya que la posición (2, 2) no es un obstáculo.

Una vez que se ha recorrido toda la matriz, el resultado se encuentra en la posición (4, 4) de la matriz, que representa el número de caminos únicos desde la esquina superior izquierda hasta la esquina inferior derecha de la cuadrícula, evitando los obstáculos.

En este caso práctico hipotético, el número de caminos únicos se desconoce, ya que no se ha resuelto el problema en sí mismo. Pero utilizando el algoritmo Unique Paths y modificando la matriz para tener en cuenta los obstáculos, se puede encontrar la solución de manera eficiente.
